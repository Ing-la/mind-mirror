# 心镜 MindMirror — 内核设计

## 层级命名

| 层级 | 单位 | 包含 | 用途 |
|------|------|------|------|
| **幕** | 最小单元 | 两个角色各发言 3 次（共 6 条消息） | 阅读节奏——给用户留一个阅读间隙 |
| **章** | 4 幕 = 1 章 | 24 条消息 + 1 次章节总结 | 技术压缩点——每章结束时 API 总结并压缩历史 |

- **幕** = 阅读的自然断点，用户看完 6 轮交锋后可以选择继续、插话或结束
- **章** = 技术上的上下文管理节点，每 4 幕做一次总结压缩
- 显示格式：**第 1 章 · 第 3 幕**

---

## 对话交互逻辑

### 核心流程

```
用户点击「召唤·让他们聊聊」
  └─→ 第 1 章 · 第 1 幕 开始
        ├─ 心声 A 发言（流式输出）
        ├─ 心声 B 发言（流式输出）
        ├─ 心声 A 发言
        ├─ 心声 B 发言
        ├─ 心声 A 发言
        └─ 心声 B 发言
      ↓ 自动停止 → 显示两个按钮：「我要发言」|「下一幕」
      
      ├─ [下一幕] → 第 1 章 · 第 2 幕（同上）
      ├─ [下一幕] → 第 1 章 · 第 3 幕（同上）
      ├─ [下一幕] → 第 1 章 · 第 4 幕（同上）
      │                ↓
      │          第 1 章结束
      │          调 API 做章节总结 + 历史压缩
      │                ↓
      ├─ [下一幕] → 第 2 章 · 第 1 幕（从压缩后的上下文开始）
      ...
```

### 详细规则

1. **每幕固定 6 条消息** — A→B→A→B→A→B，交替各 3 次
2. **每幕结束后自动暂停** — 显示两个按钮：「我要发言」|「下一幕」
3. **「我要发言」** — 用户输入自己的观点，插入对话历史，进入下一幕
4. **「下一幕」** — 直接启动下一幕，始终可用
5. **每 4 幕 = 1 章** — 章结束时 API 总结整章，压缩最早的历史
6. **「落幕」** — 用户随时结束整场对话

---

## 每章结束时的 API 调用

第 4/8/12…幕结束时触发。调用 LLM 做两件事：

1. **生成章总结** — 一段连贯的文字，概括整章 4 幕的核心发展
2. **输出结构化的立场摘要** — 每个角色的关键观点提炼

```
Prompt：请总结以下对话中两位角色的核心观点和整章进程。

{本章 24 条完整消息}

以 JSON 格式输出：
{
  "chapterSummary": "一段连贯的文字总结...",
  "positions": {
    "理性先生": ["观点1", "观点2", ...],
    "冲动崽": ["观点1", "观点2", ...]
  }
}
```

无流式、temperature 0.3、max_tokens 较小，稳定输出 JSON。

**从下一章第 1 幕起**，所有消息历史替换为：

```
messages = [
  { role: "system", content: "[角色设定 + 话题背景]" },
  { role: "system", content: "【已完成章节摘要】\n{chapter positions}" },
  // 保留最近 1 幕的完整消息以维持连贯性
  { role: "assistant", name: "理性先生", content: "..." },
  { role: "assistant", name: "冲动崽", content: "..." },
  ...
]
```

之前章的所有原始消息丢弃，只保留 JSON 摘要。

## 按钮状态机

| 状态 | 主按钮区 | 右上角按钮 |
|------|----------|------------|
| 初始（未开始） | 「召唤·让他们聊聊」 | 无 |
| 发言中 | 思考中... | — |
| 幕结束 | 「我要发言」|「下一幕」 | 无 |
| 落幕 | 无 | **「返场」** |

### 落幕与返场

- **落幕**：用户随时点击右上角「落幕」，对话标记为已结束，消息历史保留
- **返场**：落幕后右上角变为「返场」；点击回到幕结束状态，恢复「我要发言」|「下一幕」

## 前端状态管理

```typescript
status: 'idle' | 'speaking' | 'paused' | 'ended'
幕次: number              // 当前第几幕（从 1 开始）
章次: number              // 当前第几章（每 4 幕 +1）
幕内消息计数: number      // 当前幕已输出几条（0-6）
发言中: boolean           // 是否正在流式输出
消息历史: Message[]       // 完整对话记录
章节摘要: ChapterSummary[] // 每章结束后的总结
```

## Prompt 设计

### 整体架构

所有角色发言统一标为 `role: "assistant"`，通过 `name` 字段区分身份。用户插话标为 `role: "user"`。

```
POST https://api.deepseek.com/chat/completions
{
  "messages": [
    { "role": "system",    "content": "[角色设定]" },
    { "role": "system",    "content": "[章节摘要（第 2 章起）]" },
    { "role": "assistant", "name": "理性先生", "content": "..." },
    { "role": "assistant", "name": "冲动崽",   "content": "..." },
    { "role": "assistant", "name": "理性先生", "content": "..." },
    ……
  ],
  "stream": true,
  "temperature": 0.8,
  "max_tokens": 512
}
```

### System Prompt

针对当前要发言的角色动态生成。编排顺序：**你是谁 → 你的性格 → 话题背景 → 用户介入 → 对话规则**。

```
你是一个叫「{voice.name}」的虚拟角色，代表用户内心的一种声音。

## 你的性格设定
{voice.soul}

## 当前话题背景
{stage.background}

## 用户的介入
{IF 用户之前插话: 用户说：「{userMessage}」}
{ELSE: 对话尚未有用户介入。}

## 对话规则
- 你用中文说话，语气自然。
- 每条发言 2-5 句话即可。
- 你可以反驳、赞同或提出新角度。
- 如果对方说得有道理，承认它。
- 不要反复重复已充分讨论的观点。
- 自然地回应对方上一轮的核心论点。
- 不要刻意总结或结束对话。
- 不要出现"作为一个人工智能"之类的表述。
```

### 消息历史格式

```
// 正常轮流
[
  { role: "assistant", "name": "理性先生", "content": "我先算一笔账..." },
  { role: "assistant", "name": "冲动崽",   "content": "人生又不是 Excel！" },
  { role: "assistant", "name": "理性先生", "content": "后悔论是逻辑陷阱..." },
]

// 用户插话后
[
  { role: "assistant", "name": "理性先生", "content": "..." },
  { role: "assistant", "name": "冲动崽",   "content": "..." },
  { role: "user", "content": "我觉得你们说的都有道理..." },
  { role: "assistant", "name": "理性先生", "content": "用户说得对..." },
]
```

### 参数

| 参数 | 值 | 说明 |
|------|-----|------|
| temperature | 0.8 | 适当随机性 |
| max_tokens | 512 | 每条 2-5 句话 |
| stream | true | 流式到前端 |

## 数据流

```
用户点击按钮
  └─→ 前端确定当前应发言的心声 ID
  └─→ 调用 API: POST /api/chat { stage, voiceId }
  └─→ 后端构建 messages 数组：
        ├─ system: 角色设定（含用户介入段）
        ├─ system: 章节摘要（第 2 章起）
        ├─ assistant (name): 对话历史
        └─ 不含伪装 role
  └─→ 调用 DeepSeek 流式 API
  └─→ SSE → 前端逐字渲染
  └─→ 完成一条消息后：
        ├─ 保存到 localStorage
        ├─ 幕内计数 +1
        ├─ 若 < 6 → 自动触发下一个心声
        └─ 若 = 6 →
              ├─ 若幕次 % 4 === 0 → 调 API 做章节总结
              └─ 显示「我要发言」|「下一幕」
```

---

## 与当前实现的差异（需要修改的部分）

1. **消息标记** — 伪装 user/assistant → `role: assistant` + `name` 字段
2. **幕内自动轮流** — 不再手动点角色发言
3. **固定轮次** — 每幕各 3 次，共 6 条
4. **幕结束按钮** — 「我要发言」|「下一幕」
5. **落幕 / 返场** — 右上角按钮
6. **章节系统** — 每 4 幕 = 1 章，章结束时 API 总结 + 压缩
7. **第 2 章起注入章节摘要** — 替代原始历史消息
