# 心镜 MindMirror —— 应用设计文档

> 把脑袋里的小人具像化。
> 每个人心里都有一支合唱团。面对选择时，它们就开始争吵。
> MindMirror 让这些声音走出脑海，在你面前对话。

---

## 一、核心理念

### 我们在解决什么问题？

当你面临选择、感到犹豫、需要做决定的时候，脑子里总会有不同的声音在说话：

- "选这个吧，稳定，安全。"
- "可是那个才有意思啊，人生就一次。"
- "你想清楚后果了吗？"
- "别怂，试试又不会怎样。"

这些声音我们称之为「心音」。它们是我们内心不同侧面的投射——谨慎的、冲动的、理性的、感性的、乐观的、悲观的。

**MindMirror 的作用**：把这些模糊的内心对话，变成两个（或多个）具象化的「小人」，让它们真正开口说话、互相辩论、彼此启发。用户不再是脑子里一团乱麻，而是像一个温柔的倾听者，看着自己的不同侧面在小声地聊天。

### 一句话描述

> 一个轻量级的「内心对话」小天地 —— 你告诉它你在纠结什么，它让两个小人替你聊聊清楚。

---

## 二、核心概念

### 2.1 小人 (Voice)

每个「小人」代表用户内心的一种声音倾向。它有三个核心属性：

- **名字**：比如"理性哥"、"冲动崽"、"怂包酱"
- **Soul（灵魂设定）**：一段文字描述这个小人是什么性格、说话风格、价值观倾向
- **声音 / 头像**：视觉上让小人更鲜明

用户也可以使用预置的小人模板：

| 预设小人 | Soul |
|---------|------|
| 🧊 理性先生 | 冷静、逻辑、数据驱动，擅长分析利弊 |
| 🔥 冲动的伊格 | 热情、直觉、活在当下，讨厌后悔 |
| 🛡️ 保守阿姨 | 安全第一，凡事往坏处想，防范于未然 |
| 🌈 梦想家露娜 | 理想主义，相信可能性，不被现实束缚 |

### 2.2 对话场景 (Stage)

一次内心对话的单元，包含：

- **题目**：用户在纠结什么？（如"要不要辞职去创业？"）
- **背景**：补充信息（如"我 28 岁，有 10 万存款，工作 3 年..."）
- **角色**：参与对话的小人（至少 2 个，最多 3 个）
- **对话流**：小人们的对话记录

## 三、用户体验流程

### Step 1：开始一次对话

用户打开应用，看到温暖的起始页。输入：

```
题目：要不要买那辆摩托车？
背景：我攒了两年钱，但家里人说太危险。朋友说趁年轻就该疯一把。
```

### Step 2：选小人（可选）

系统根据题目自动推荐两个小人。用户也可以手动替换：

```
系统推荐：
  🧊 理性先生  ←→  🔥 冲动的伊格

用户可以选择：
  - 接受推荐
  - 换掉其中一个（从预设库选择）
  - 自定义一个小人（写自己的 Soul）
```

**极简模式（默认）**：不用选人，默认就是「理性先生 vs 冲动崽」，系统根据题目自动分配初始立场。

### Step 3：对话开始

两个小人开始围绕题目自由对话。对话是**自动流动**的，不需要用户逐一点击。节奏适中，每条消息中等长度。

```
🧊 理性先生：
"我先算一笔账。摩托车 3 万，保险+头盔+装备 5000，
养车一年大概 3000。你存了两年才 5 万，
花掉大半换一个使用频率不高的玩具，划算吗？"

🔥 冲动的伊格：
"拜托，人生又不是 Excel 表格！
你存钱是为了什么？不就是想过得开心吗？
每次你说'再等等'，时间就溜走了一点。
你告诉我，如果八十岁回头看，你会后悔没买吗？"

🧊 理性先生：
"后悔论是逻辑陷阱。用'未来会后悔'合理化冲动的决策，
等于承认现在的自己无法对未来的自己负责。
我们可以换一个框架：这 3 万如果用来学新技能，
会不会带来更长远的回报？"
```

### Step 4：用户介入（可选）

用户在对话过程中可以：

- **插话**：在输入框打字，插入自己的观点，小人们会据此调整方向
- **@引导**：用 `@理性先生 他说的有道理吗？` 来指定小人回应
- **喊停换题**：有时候聊跑偏了，用户可以重设方向

### Step 5：收拢与回顾

对话自然结束（用户觉得聊够了），系统提供：

- **对话摘要**：每个小人的核心观点提炼
- **冲突地图**：双方在哪些点上达成一致，哪些点仍有分歧
- **一句话台词**：最有启发性的一句话高亮
- **继续 / 再来一次**：用户可以调一个参数重新开始（比如换一个小人）

## 四、技术架构

### 4.1 总体架构

```
┌─────────────────┐      ┌────────────────────────┐      ┌─────────────┐
│  前端 (Next.js)   │─────▶  心音引擎 (Vercel Functions) │─────▶  LLM 适配层  │
│                  │◀────│  Orchestrator           │◀────│              │
└─────────────────┘      └────────────────────────┘      └─────────────┘
                             │
                      ┌──────┴──────┐
                      │  存储 (localStorage)│
                      └─────────────┘
```

**轻量级选择**：Next.js + Vercel Serverless Functions，零服务器运维，开箱即用。

### 4.2 多 LLM 编排核心 (心音引擎)

这是这个 app 的技术灵魂。核心逻辑是**让两个 LLM 实例带着不同的角色设定进行多轮对话**。

```
循环流程：

1. [系统] 构建 User A 的 System Prompt（名字 + Soul + 当前题目 + 背景）
2. [系统] 构建 User B 的 System Prompt（名字 + Soul + 当前题目 + 背景）
3. [系统] 将对话历史 + A 的个性 → LLM_A → 生成 A 的发言
4. [系统] 将对话历史 + B 的个性 → LLM_B → 生成 B 的发言
5. 重复 3-4，直到达到结束条件
```

**关键设计决策**：

- **每个小人独立调用 LLM**：不是用一个 prompt 让 LLM 扮演两个角色（那样会精神分裂且缺乏真正的对抗性），而是两个独立的 LLM 调用，**各自只知道自己的角色和完整的对话历史**。这样产生的对话才真实。
- **同一个 API 但不同 System Prompt**：两个小人可以挂同一个 LLM 提供商（比如都走 DeepSeek），只是 System Prompt 不同。高阶用户可以给不同小人配置不同的 LLM（比如一个 GPT-4 一个 Claude）。
- **流式输出**：每条发言生成时流式输出到前端，就像真的在听一个人说话一样。

### 4.3 对话管理

```
数据结构（简化）：

Stage {
  id: string
  title: string          // 题目
  background: string     // 背景信息
  voices: Voice[]        // 参与的小人
  messages: Message[]    // 对话记录
  status: 'ongoing' | 'paused' | 'ended'
  created_at: timestamp
}

Voice {
  id: string
  name: string
  soul: string
  avatar?: string
  provider?: string      // LLM 配置（可选，默认走全局）
  brainId?: string
}

Message {
  role: 'assistant'      // 小人的发言
  voiceId: string
  content: string
  turn: number
  tokenCount?: number
  timestamp: timestamp
}
```

### 4.4 LLM 调用策略

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| 轮流发言 | A→B→A→B，固定顺序 | 默认模式 |
| 主动抢话 | LLM 决定是否回应，可以跳过或连续说 | 进阶模式（需要额外判断调用） |
| 用户插话 | 用户发言后，指定或自动选一个小人回应 | 用户介入 |

**结束条件**（满足其一即可）：
- 双方达成共识（LLM 输出检测到"同意对方"）
- 重复轮回（连续 3 轮没有新论点）
- 对话轮次达到上限（默认 10 轮）
- 用户手动结束

## 五、用户界面设计

### 设计原则

- **温暖纸感**：暖白底色、柔和阴影、圆润边角，像一本摊开的精装笔记本
- **亲近感**：不设"舞台"式的距离，让界面像一个可以安心袒露内心的角落
- **文字优先**：文字是主角，排版疏朗通透，给思绪呼吸的空间
- **陪伴感**：小精灵（头像）温柔浮动，色彩柔和，没有攻击性

### 界面示意

```
┌──────────────────────────────────────┐
│  🤔 要不要买那辆摩托车？            │
│  背景：攒了两年钱...                 │
│                                      │
│     🧊             🔥               │
│   理性先生       冲动的伊格          │
│                                      │
│  ┌────────────────────────────────┐  │
│  │ 🧊 理性先生                    │  │
│  │ 我先算一笔账...                │  │
│  └────────────────────────────────┘  │
│                                      │
│  ┌────────────────────────────────┐  │
│  │ 🔥 冲动的伊格                 │  │
│  │ 人生又不是 Excel 表格！        │  │
│  └────────────────────────────────┘  │
│                                      │
│  ┌────────────────────────────────┐  │
│  │ 💬 我也想说说...              │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

### 三种视图模式

1. **对话视图**（默认）：上下滚动的对话流，类似聊天界面
2. **分栏视图**：左右分栏，像两个人在面对面交谈（适合观点交锋时）
3. **总结视图**：对话结束后，提炼核心观点

## 六、MVP 范围（第一版做什么）

### 必须做

- [x] 用户输入题目和背景
- [x] 2 个小人自动对话（轮流发言）
- [x] 3 个预置小人模板（用户可以直接选）
- [x] 自定义小人（设定名字和 Soul）
- [x] 流式输出（打字机效果）
- [x] 对话历史保存（浏览器本地存储）
- [x] 用户插话功能
- [x] 对话结束后的摘要

### 不做（第一版）

- [ ] 3 个小人同时对话
- [ ] 每个小人配置不同的 LLM 提供商
- [ ] 抢话模式（主动回应）
- [ ] 声音朗读
- [ ] Web 多人共享
- [ ] 移动端适配（Web 已自动支持基础适配，后续优化）

## 八、命名与品牌

**应用名候选**：
- **心镜 MindMirror** —— 内心的镜子，照见自己的声音
- **心音 Heartalk** —— 内心的话语
- **脑内心声 Mindwhisper** —— 内心的低语

**推荐**：**MindMirror / 心镜**
> 当你面对镜子，看到的是自己。当面对 MindMirror，看到的是心里不同的自己。
