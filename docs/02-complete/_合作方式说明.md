# 🤝 我们的合作方式

> 这个文件夹（`docs/02-complete/`）全权由 AI 负责管理。
> **你的任务**：提供原始素材（截图、文案想法）。
> **我的任务**：整理、分类、命名、写成参赛级别的展示内容。

---

## 一、工作流总览

```
你 → ① 把截图放到 raw/ 文件夹
    → ② 告诉我：这张图是什么内容、对应什么场景
    → ③ 如果有文案灵感，直接告诉我

我 → ① 从 raw/ 取图 → 裁剪/重命名 → 归档到 screenshots/ 对应子目录
    → ② 将图片链接嵌入 materials/ 中的对应 markdown
    → ③ 完成文字润色、结构组织 → 最终合成参赛帖草稿
```

---

## 二、具体分工

### 你负责的事（很简单）

| # | 你要做的事 | 怎么做 |
|---|-----------|--------|
| 1 | **截图** | 打开 MindMirror 或 TRAE IDE，截取关键画面 |
| 2 | **丢进 raw/** | 把截图文件放到 `docs/02-complete/raw/` 目录下 |
| 3 | **告诉我一声** | 说一句如 *"我放了 3 张截图，分别是首页、对话页面和 TRAE 开发界面"* |
| 4 | **补充说明** | 如果截图有特殊背景（比如这张图展示了一个 bug 修复过程），告诉我 |
| 5 | **提供 Session ID** | 打开 TRAE IDE，双击对话任务复制 Session ID，发给我 |

### 我负责的事

| # | 我要做的事 | 说明 |
|---|-----------|------|
| 1 | **研究比赛要求** | ✅ 已研究完毕（见 `_材料总清单.md`） |
| 2 | **分类整理截图** | 从 raw/ 取图 → 重命名 → 放入 `screenshots/` 对应子目录 |
| 3 | **撰写文字内容** | 按比赛的 4 个评审维度撰写参赛级文案 |
| 4 | **合成参赛帖** | 最终输出一篇可直接发布的 Demo 作品帖 |
| 5 | **维护材料清单** | 跟踪哪些材料已就绪、哪些还缺 |

## 四、截图命名规则

你放到 `raw/` 的文件名随意，我来统一命名。命名格式如下：

```
{序号}-{场景描述}-{视图}.{ext}
```

例如：

| 你丢进来的（raw/） | 我整理后的（screenshots/） |
|-------------------|--------------------------|
| `截图1.png` | `01-homepage-full.png` |
| `图片2.jpg` | `02-dialogue-stage.png` |
| `QQ截图123.png` | `03-trae-chat-session.png` |

## 六、一次典型协作场景

**场景：** 你需要我帮忙写好"TRAE 实践过程"这部分内容

1. 你打开 TRAE IDE，截 3-4 张关键步骤的图
2. 把截图放到 `raw/` 文件夹
3. 你跟我说：*"raw 里放了 4 张图，分别是：(1) 我用 TRAE 写第一个 prompt 生成对话引擎 (2) TRAE 生成的代码跑起来的效果 (3) 修改角色性格的过程 (4) 部署到 Vercel 成功"*
4. 同时把对应的 **Session ID** 发给我（双击 TRAE 对话复制）
5. 我开始工作：
   - 把 4 张图从 raw/ 取出，重命名后放入 `screenshots/02-trae-process/`
   - 打开 `materials/03-trae-practice.md`，嵌入图片并撰写文字
   - 补充 Session ID
6. 完成后我会告诉你：*"TRAE 实践部分已写好，你去看看效果？"*
