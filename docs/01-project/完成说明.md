## 已实现功能

### 1. 首页 (`/`)
- **品牌展示**：心镜 MindMirror 标题 + 副标题，暖色调渐变顶栏
- **历史对话列表**：从 localStorage 读取所有已保存的对话，展示题目、状态（进行中/已结束）、消息数量、参与角色
- **空状态引导**：无记录时显示引导文案
- **开始新对话**：快速入口按钮
- **删除对话**：每个卡片上可删除指定对话
- **状态同步**：跨标签页数据同步（`storage` 事件监听）

### 2. 新建对话 (`/stage/new`)
- **题目输入**：核心问题输入框（必填）
- **背景输入**：补充信息文本框（可选）
- **角色选择**：
  - 两个角色槽位，默认「理性先生 vs 冲动的伊格」
  - 点击任一槽位弹出选择面板
  - 4 个预设小人可选：理性先生、冲动的伊格、保守阿姨、梦想家露娜
  - 支持自定义小人（自定义名字 + Soul 性格描述）
- **幕启**：保存到 localStorage 后跳转到对话剧场

### 3. 对话剧场 (`/stage/[id]`)
- **题目 & 角色展示**：顶部显示题目、背景、两个角色卡片（带渐变背景色）
- **流式输出**：调用 DeepSeek API 进行流式对话，逐字显示（打字机效果 + 闪烁光标）
- **半自动对话流程**：
  - 初始状态 → 点击「幕启·开始对话」→ 第一个小人发言（流式）
  - 完成后 → 点击「让 XX 发言」→ 第二个小人发言
  - 交替进行，每轮手动触发
  - 最多 10 轮自动截停
- **用户插话**：对话过程中可输入文字插话，插入后触发当前应发言的小人回应
- **结束对话**：可手动结束，或达到最大轮次后自动提示
- **观点摘要**：对话结束后自动提取双方前 3 条发言的核心内容作为摘要
- **对话历史**：每条消息保存至 localStorage，刷新页面不丢失

### 4. API 路由 (`/api/chat`)
- **POST 接口**：接收 `{ stage, voiceId }`，返回流式文本
- **DeepSeek 集成**：调用 DeepSeek Chat API，支持 `stream: true`
- **System Prompt 构建**：根据角色设定（名字 + Soul + 题目 + 背景）生成个性化系统提示词
- **对话历史注入**：将已有消息作为上下文传入，保持对话连贯性
- **SSE 转流式**：将 DeepSeek 的 SSE 响应转换为纯文本流

## 技术架构

### 项目结构

```
src/
├── app/
│   ├── api/chat/route.ts      # DeepSeek API 流式代理
│   ├── stage/
│   │   ├── new/page.tsx       # 新建对话页
│   │   └── [id]/page.tsx      # 对话剧场页
│   ├── globals.css            # 全局样式（MindMirror 暖色调主题）
│   ├── layout.tsx             # 根布局
│   └── page.tsx               # 首页
└── lib/
    ├── types.ts               # 数据模型定义
    ├── store.ts               # localStorage 操作
    ├── preset-voices.ts       # 预设角色模板
    └── llm.ts                 # DeepSeek API 调用封装
```

### 数据模型

```typescript
interface Voice {
  id: string
  name: string
  soul: string        // 性格设定
  emoji: string       // 表情符号
  color: string       // 'cool' | 'warm' | 'green' | 'yellow'
}

interface Message {
  role: 'assistant' | 'user'
  voiceId?: string
  content: string
  turn: number
  timestamp: number
}

interface Stage {
  id: string
  title: string
  background: string
  voices: Voice[]
  messages: Message[]
  status: 'ongoing' | 'ended'
  createdAt: number
}
```

### 预设角色

| 角色 | Emoji | 性格 |
|------|-------|------|
| 理性先生 | 🧊 | 冷静、逻辑、数据驱动 |
| 冲动的伊格 | 🔥 | 热情、直觉、活在当下 |
| 保守阿姨 | 🛡️ | 安全第一，防范于未然 |
| 梦想家露娜 | 🌈 | 理想主义，相信可能性 |

### 对话引擎逻辑

1. 每个角色拥有独立的 System Prompt（基于名字 + Soul + 题目 + 背景）
2. 调用 DeepSeek API 时传入完整的对话历史作为上下文
3. 两个角色交替发言，各自独立调用 API
4. 流式输出解析 SSE 数据包，逐字推送到前端
5. 温度 `0.8`，最大 Token `512`，保证每轮发言自然且适度

## 使用方式

### 1. 配置 API Key

复制 `.env.local.example` 为 `.env.local`，填入你的 DeepSeek API Key：

```bash
cp .env.local.example .env.local
# 编辑 .env.local，填入你的 Key
DEEPSEEK_API_KEY=sk-your-api-key-here
```

> 从 [platform.deepseek.com](https://platform.deepseek.com/) 注册获取 API Key

### 2. 启动开发服务器

```bash
npm run dev
```

### 3. 打开浏览器

访问 `http://localhost:3000` 即可使用。

## 当前限制 & 后续计划

### 当前限制
- **需要 DeepSeek API Key**：暂不支持用户自行配置其他 API
- **无用量限制**：尚未实现每日调用次数限制
- **摘要简单**：当前摘要仅截取前几条消息，未使用 LLM 进行智能总结
- **无登录系统**：数据仅存于浏览器 localStorage，清除浏览器数据会丢失
- **移动端适配基础**：支持响应式布局，但未做精细化移动端优化

### 用户提到的后续计划
- [ ] 每日 API 调用次数限制
- [ ] 用户自行配置 API Key
- [ ] 数据库存储（替代 localStorage）
- [ ] 更完善的 UI/UX 设计迭代
- [ ] 移动端精细化适配
